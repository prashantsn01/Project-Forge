# 🔥 ProjectForge AI — Student Project Builder (Fullstack)

> **AI-powered fullstack app** that generates complete, runnable code for any student project — built on your HTML frontend + a Node.js backend that calls the Anthropic Claude API.

---

## Architecture

```
projectforge/
├── backend/          ← Node.js + Express server
│   ├── server.js     ← Main server: /api/generate + /api/download
│   ├── package.json
│   └── .env.example  ← Add your Anthropic API key here
│
└── frontend/
    └── index.html    ← Your full frontend (no build step needed)
```

---

## Quick Start

### 1. Set up the backend

```bash
cd backend
npm install
cp .env.example .env
```

Edit `.env` and add your Anthropic API key:
```
ANTHROPIC_API_KEY=sk-ant-your-key-here
```

Get your key at: https://console.anthropic.com

```bash
npm run dev        # starts on http://localhost:3001
```

### 2. Open the frontend

Just open `frontend/index.html` in your browser — no build step needed.

Or serve it with VS Code Live Server, or any static file server:
```bash
# option: serve with npx
npx serve frontend
```

### 3. Connect & generate

1. Open `frontend/index.html`
2. The **Backend URL** field defaults to `http://localhost:3001` — click **Connect**
3. Describe any project → pick stack → select features → click **⚡ Generate My Project with AI**

## How It Works

```
Student types project idea
        ↓
Frontend sends POST /api/generate to backend
        ↓
Backend calls Anthropic Claude claude-opus-4-5 API with detailed prompt
        ↓
Claude generates complete, custom code for the exact project described
        ↓
Backend parses JSON response → returns structured file tree
        ↓
Frontend renders IDE viewer, file tree, code preview, insights
        ↓
Student clicks Download ZIP → backend streams archiver ZIP
```

For every project the AI writes:

| File Type | What's Inside |
|-----------|--------------|
| Backend models | Domain-specific schemas (e.g. for Hospital: `Patient.js`, `Doctor.js`, `Appointment.js`) |

| Business routes | CRUD with real domain logic — not generic Item.js |
| Frontend | Real components named after the project domain |

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/health | Server health check |
| POST | /api/generate | Generate project via AI |
| POST | /api/download | Stream ZIP of generated files |

### POST /api/generate

**Request:**
```json
{
  "description": "A hospital appointment system...",
  "stack": "React + Node.js",
  "features": ["User Authentication", "Admin Dashboard", "Email Notifications"],
  "level": "3",
  "commentMode": "standard"
}
```

**Response:**
```json
{
  "ok": true,
  "project": {
    "projectName": "MediCore",
    "description": "Hospital appointment booking system",
    "stack": "React + Node.js",
    "totalLines": 1240,
    "folders": [...],
    "insights": [...],
    "setupSteps": [...]
  }
}
```

### POST /api/download

**Request:**
```json
{ "project": { ...generatedProject } }
```

**Response:** `application/zip` file stream

## Deployment

### Backend (Railway / Render / Heroku)

```bash
# Set these environment variables on your platform:
ANTHROPIC_API_KEY=sk-ant-...
PORT=3001
CLIENT_URL=https://your-frontend-domain.com
```

### Frontend (Netlify / Vercel / GitHub Pages)

Update the **Backend URL** in the app to point to your deployed backend.
Or set it as a default in `frontend/index.html`:
```js
var BACKEND_URL = 'https://your-backend.up.railway.app';
```

## Tech Stack

- **Frontend:** Vanilla HTML/CSS/JS (your original design, preserved exactly)
- **Backend:** Node.js + Express
- **AI:** Anthropic Claude claude-opus-4-5 via REST API
- **ZIP:** `archiver` npm package
- **Auth:** None (students interact directly — add auth if deploying to production)
